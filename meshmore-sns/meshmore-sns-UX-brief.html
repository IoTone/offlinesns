Meshmore SNS — UX Design Brief · 6 concepts

Within the stated intent (Wipeout 2097 × Neon Genesis Evangelion; dashboard-home; hyperlocal off-grid mesh; R1–R13). Each ships a matching audible theme pack with mandatory visual + haptic parity (hearing-impaired-first). This page is fully static HTML — it renders with zero JavaScript; the optional audio-preview buttons are progressive enhancement only.

Indicative mockups (system fonts). Real OSS, license-clean type plan: Saira / Saira Condensed (Eurostile-class display), Departure Mono (bitmap console telemetry), M PLUS 1 Code (mono + full Japanese). Eurostile / Matisse / Helvetica = inspiration, never bundled. Sourced research + licensing table: meshmore-sns-UX-brief.md.

Visual language

  • Telemetry-grid dashboard — everything visible at once
  • Chamfered panels, diagonal warning-stripe headers, CRT scanline; emergency = colour-invert to red + faster scanline
  • Departure Mono telemetry; Saira ALL-CAPS headers; slashed-zero numerals (the Eva "SOUND ONLY" motif)

Audible pack — "Mission Control"

Austere/synthetic: square-wave blips, terse 2-tone message, Geiger-ish node ticks, descending sawtooth klaxon for critical. TTS voice: flat, clipped, "operator".
Parity: every cue → warning-stripe flash + a distinct haptic pattern (single / double / long-long-long).

Audible pack — "Velocity"

Arcade/electronic: triangle arpeggios, rising whoosh on navigation, major-third confirm, energised alert sweep. TTS voice: upbeat, mid-tempo.
Parity: swipe-sweep is also the visual transition; alerts = amber band + haptic burst.

Visual language

  • Node radar is the hero (→ F1 live-map future)
  • Chat as a draggable Material sheet (peek/half/full = R6)
  • Lower noise, AA+ contrast, large targets, reduce-motion aware

Audible pack — "Sonar"

Calm/organic: soft sonar ping for node discovery (mirrors the radar), gentle two-note message, low calm link tones, sparse — field-friendly under noise. TTS voice: calm, natural.
Parity: ping = radar-blip pulse; link tones = status-rail colour + soft haptic.

Palette

ink cream SEELE red bone line

Visual language

  • One huge numeral per screen; ruthless hierarchy, near-zero chrome
  • Monolith bars (SEELE "SOUND ONLY"); status = full-width slab, colour-inverts to red on alert
  • Highest legibility/accessibility of all six; trivial under sun and at large text scale

Audible pack — "Tribunal"

Severe and sparse: deep resonant sub-bass strikes, single low choral swell for critical, almost silent otherwise. TTS voice: slow, grave, committee.
Parity: the status slab is the visual cue (slab → red) + a single firm haptic; never relies on the tone.

Palette

ink magenta acid cyan paper

Visual language

  • Flat colour-block layout; each channel gets a fake corporate "sub-brand" (the Wipeout team-livery system, applied to channels)
  • Kanji/katakana as graphic elements; coded labels (PHASE_02); loud, playful, rave-era TDR
  • Highest visual energy; needs care on contrast (handled via the parity rules — colour never sole carrier)

Audible pack — "Pure Phase"

Bright syncopated arcade blips, funky confirms, Wipeout-soundtrack energy. TTS voice: punchy, stylised.
Parity: each event also flips its sub-brand block + a rhythmic haptic; loudness never required.

Palette

oled amber dim alert panel

Visual language

  • Map/bearing-first; waypoints with range + heading; sweep line
  • Single-hue amber phosphor on true black — OLED power-frugal, dark-adapted (night-vision friendly), high glare resistance
  • Codec/recon comms styling; gloves-grade targets; alt phosphor-green theme variant

Audible pack — "Codec"

Comms-radio: terse UHF beeps, brief squelch on link change, the codec-call ring for critical. Low volume, cuts through ambient noise. TTS voice: clipped radio-operator.
Parity: amber blink on the map/log + terse haptic; designed to also work fully silent (field stealth).

Notes

  • Device config drives the real M4/R7 protocol surface already built in packages/meshcore (set radio params / TX power / advert name+latlon / channels + PSK / other params / tuning; reads DEVICE_INFO, battery, time).
  • App settings: BLE auto-reconnect (the M7 backoff policy), language (R4), TTS off-by-default (R5), critical→system-notification parity, the M6 diagnostics export, About/Terms (R9/R10).
  • Profile & Personalization (R14): theme presets = the six concepts (+ D high-contrast, F night) as selectable themes, a custom palette editor, font family + size scale, density, per-event audio toggles + pack, and the accessibility switches incl. the one-tap “visual + haptic only” mode. Persisted via shared_preferences.
  • These three sit under the Settings primary view (pushed via go_router), not as new swipe pages — they inherit the chosen concept's skin.
